# Changelog

Все существенные изменения проекта фиксируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
версионирование — на [Semantic Versioning](https://semver.org/lang/ru/).

## [2.1.0-beta] - 2026-07-12

### Added

- Отображение активной катушки в статусе печати: цвет (эмодзи) и позиция слота (`T1`-`T4`, `A`-`D`) для Creality CFS и Bambu AMS в едином формате `[-emoji--] T1B PLA`. Без AMS/CFS или при печати с внешнего держателя — отдельная строка с материалом/цветом внешнего прутка, если он известен.

### Changed

- Процент прогресса для Moonraker-принтеров считается от `virtual_sdcard.progress` (позиция в файле) вместо `display_status.progress` (оценка слайсера по времени, уходит на печатях с паузами на смену прутка).
- Логика Bambu/Moonraker вынесена из `bot.py` в отдельные модули-коннекторы (`connectors/`) за общим интерфейсом `PrinterConnector` (`connectors/base.py`) и типами `PrinterState`/`SpoolInfo` (`models.py`) — чтобы добавить поддержку нового принтера или мультиматериальной системы, достаточно положить новый файл в `connectors/`, ничего в ядре редактировать не нужно (см. README, раздел «Поддержка нового оборудования»). Внешнее поведение и формат сообщений не изменились.

### Note

- Логика для Bambu AMS основана на публично известной схеме MQTT (`tray_now`), а не на тестах с реальным железом — требует проверки, когда AMS будет физически подключена.

## [2.0.1] - 2026-07-11

### Fixed

- Дублирование сообщений с прогрессом печати из-за ошибочного сброса `progress_message_id` на сетевых ошибках и невозможности редактировать подпись у текстовых сообщений. Прогресс теперь всегда фото: последний снимок камеры для текущей печати или заглушка.

## [2.0.0] - 2026-07-05

### Added

- Поддержка произвольного числа принтеров любого из двух типов через YAML-манифест `printers.yaml` (шаблон — `printers.example.yaml`). Имя принтера из манифеста используется как тег в сообщениях (`[name] ...`) и как суффикс команд.
- Поддержка принтеров на Klipper/Moonraker (класс `MoonrakerPrinterMonitor`) наравне с Bambu Lab: статус и снимок камеры получаются через REST API Moonraker (`/printer/objects/query`, `camera_snapshot_url`).
- Опциональное поле `api_key` для Moonraker-принтеров — передаётся заголовком `X-Api-Key`, если Moonraker закрыт авторизацией.
- `/add_printer` — интерактивный диалог в Telegram: выбор типа принтера, последовательный ввод параметров, тест подключения, и при успехе — добавление принтера в мониторинг без перезапуска бота, с сохранением записи в `printers.yaml`.
- `/list_printers` — список принтеров, находящихся под мониторингом, с указанием типа.
- Команды `/status_<name>`, `/photo_<name>` для каждого принтера из манифеста; `/light_on_<name>`, `/light_off_<name>` — для принтеров Bambu. `/status` без аргументов показывает сводку по всем принтерам сразу.

### Changed

- Конфигурация принтеров перенесена из `.env` в `printers.yaml`. В `.env` остались только параметры, общие для всего бота: токен и chat_id Telegram, локаль, часовой пояс, дефолтные интервалы опроса (`DEFAULT_POLL_INTERVAL_SECONDS`, `DEFAULT_PROGRESS_UPDATE_SECONDS`).
- Команды `/light_on` и `/light_off` переименованы в `/light_on_<name>` и `/light_off_<name>` в связи с поддержкой нескольких принтеров одного типа (несовместимо с версией 1.0.0).
- `/start` формирует список доступных команд динамически, на основе принтеров, фактически загруженных на момент вызова.

### Fixed

- Синхронные вызовы `bambulabs_api` (чтение статуса, снимок камеры, управление светом, тест подключения при добавлении принтера) переведены на выполнение в отдельном потоке (`asyncio.to_thread`). Ранее они выполнялись прямо в корутинах на общем event loop: подвисание MQTT-вызова к одному принтеру (Bambu A1) блокировало весь процесс целиком, включая опрос других принтеров (Moonraker) и приём команд Telegram, на неопределённое время — до отпускания блокировки события резервировались и приходили все сразу.

## [1.0.0] - 2026-07-04

### Added

- Первая версия бота: мониторинг одного принтера Bambu Lab A1 по MQTT (`bambulabs_api`).
- Уведомления о смене статуса печати (старт, пауза, возобновление, завершение, ошибка) со снимком камеры.
- Периодическое редактируемое сообщение с прогрессом печати (файл, процент, слой, оставшееся время).
- Команды `/status`, `/photo`, `/light_on`, `/light_off`.
- Локализация сообщений (`ru`/`en`).
